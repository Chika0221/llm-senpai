# 先輩質問システム（Human as LLM）開発ロードマップ

`docs/specification.md` の要件に基づき、ユーザー主導・AI（私）サポート型で進めるためのロードマップです。

## フェーズ 1: 環境構築・基盤セットアップ
- [x] **リポジトリ初期化**: Gitリポジトリの作成とパッケージマネージャ（npm/yarn/pnpm）の設定
- [x] **バックエンド環境構築**: Node.js + Hono のプロジェクト立ち上げ
- [x] **フロントエンド環境構築**: Next.js (または React) プロジェクトの立ち上げ
- [x] **Linter/Formatter設定**: ESLint, Prettier の導入
> **AIサポート**: 最適なフレームワーク選定の相談、`package.json` の初期構成やTypeScript設定ファイルの提供

## フェーズ 2: データベース設計と構築
- [ ] **ORマッパーの導入**: Prismaの設定 (`schema.prisma` の作成)
- [ ] **DB環境の用意**: PostgreSQL環境の構築（ローカルDockerまたはSupabase等）
- [ ] **テーブル設計**: 「セッションID」「リクエスト元」「DiscordメッセージID」「ステータス」などを管理するテーブルの作成
- [ ] **マイグレーションの実行**: データベースへのスキーマ反映
> **AIサポート**: `schema.prisma` のモデリング提案、Docker Composeファイルの作成

## フェーズ 3: Discord Botの実装
- [ ] **Botの作成**: Discord Developer Portalでのアプリ作成とTokenの取得
- [ ] **Botの組み込み**: `discord.js` を用いたバックエンドへのBot連携
- [ ] **メッセージ送信機能**: 指定チャンネル（またはロール）へ質問を投稿する機能
- [ ] **リプライ検知機能**: 先輩が回答（リプライ）した内容をフックして受け取る機能
> **AIサポート**: `discord.js` の初期ボイラープレート生成、権限(Intents)設定のトラブルシューティング

## フェーズ 4: バックエンド中核機能 (OpenAI互換APIとSSE)
- [ ] **APIエンドポイント実装**: `/v1/chat/completions` のモックエンドポイント作成
- [ ] **SSE(Server-Sent Events)の実装**: HTTPタイムアウトを防ぐためのストリーミングレスポンス処理
- [ ] **ルーティングロジック**: API経由の質問をDBに保存し、Discordへ流す処理の統合
- [ ] **回答返却ロジック**: Discordでのリプライ検知後、待機中のSSEストリームへJSON形式で回答を流し込む処理
> **AIサポート**: SSEのタイムアウト対策やダミーパケット送信の実装例、Honoでの非同期ストリーム処理の提示

## フェーズ 5: フロントエンドWeb UIとWebSocket通信
- [ ] **UIのベース作成**: チャット形式で質問を入力できる画面の構築
- [ ] **WebSocket設定**: Socket.io (または同等の技術) を用いたバックエンドとの双方向通信の確立
- [ ] **質問送信とUI更新**: 画面から質問を送り、ページ更新なしで先輩の回答（Discord経由）を表示する処理
> **AIサポート**: Socket.ioのクライアント/サーバー実装例の提供、美しいチャットUIのコンポーネントコード（Tailwind CSS等）の生成

## フェーズ 6: LLM案内係（自動ルーティング）の実装
- [ ] **LLM APIの導入**: Gemini FlashやOpenAI APIなどをバックエンドから呼び出す設定
- [ ] **プロンプトエンジニアリング**: 質問内容から言語/フレームワークを特定し、JSONで返すプロンプトの作成
- [ ] **Discordへの振り分け**: LLMの判定結果に基づき、対応するDiscordチャンネルや先輩ロール（`@React先輩`など）を動的に切り替える処理
> **AIサポート**: 軽量・高速に判定するためのLLMプロンプトの設計支援

## フェーズ 7: 結合テスト・運用基盤
- [ ] **E2Eテスト**: AIエディタ（Cursor等）やWeb UIから質問し、Discordで返信してアプリに反映されるか一連の流れをテスト
- [ ] **エラーハンドリング**: Discord側でのタイムアウト、先輩が答えなかった場合のフォールバック処理
- [ ] **デプロイ**: Vercel(フロント) や Render/Railway(バックエンド)、マネージドDBへの公開
> **AIサポート**: Cursorの設定方法（カスタムエンドポイントの向き先）、デプロイ時の環境変数設定やCORSエラーの解決
