# 先輩質問システム（Human as LLM） 利用マニュアル

このドキュメントでは、本システムを「質問する側（後輩）」と「回答する側（先輩）」それぞれの視点からどのように利用するかを解説します。

---

## 👨‍💻 後輩（質問者）向けの使いかた

AIエディタ（Cursor、Copilotなど）の「カスタムAIモデル」として先輩を呼び出し、エディタ上から直接質問することができます。

### 1. エディタの設定（例：VSCode）
1. VSCodeの右側のチャット画面を開きます。
2. 下の **モデルの管理...** を探しクリック
3. **モデルの追加 → Custom Endpoint → グループ名：Horizon → APIKEY：horizon → chat-completions** 
4. エディタが出てくるので以下のように設定
   ```json
   {
		"name": "Horizon",
		"vendor": "customendpoint",
		"apiKey": "${自動入力}",
		"apiType": "chat-completions",
		"models": [
			{
				"id": "senpai1",
				"name": "LLMSenpai 1.0",
				"url": "http://localhost:7070/v1",
				"toolCalling": true,
				"vision": true,
				"maxInputTokens": 128000,
				"maxOutputTokens": 16000
			}
		]
	}
   ```


### 2. 質問のしかた
普段AIに質問するのと同じ感覚で、チャット欄に質問を入力してください。
- 例: 「このファイルの `npx prisma db push` でエラーが出ます。どうすればいいですか？」
- 例: 「この関数のリファクタリングをお願いします。」

**特徴**
- 質問を送信すると、バックエンドを通じて自動的に適切な先輩が常駐するDiscordチャンネルへ転送されます。
- 先輩が回答するまでの間、エディタはローディング状態（思考中）になります。
- 先輩からの回答は、まるでAIが生成したかのようにエディタ上に直接表示されます。

### 3. コマンドの自動実行
先輩がトラブルシューティングのためにコマンド（PowerShell等）の実行を指示した場合、エディタがそのコマンドを検知し、**あなたのローカルシェルで自動的にコマンドを実行**します。
実行結果は再び先輩に送信され、先輩がエラーの原因を特定するのに役立ちます。

---

## 🦸‍♂️ 先輩（回答者）向けの使いかた

後輩からの質問は、普段お使いの **Discord** にBot経由で通知されます。専用のアプリを開く必要はありません。

### 1. 質問の受信とスレッド
1. 後輩がエディタから質問を送信すると、特定のチャンネル（例: `#質問対応`）にBotがメッセージを投稿します。
2. 同時に、その質問専用の **「スレッド」** が自動作成されます。
3. 質問に回答する場合は、必ずその **「スレッド内」で発言** してください。（スレッド内で発言することで、システムが「その後輩への返信」として正しくルーティングします）

### 2. 通常の回答（テキスト）
スレッド内で普通にテキストを送信すると、その後輩のエディタ画面にそのままテキストが届きます。
- 例: 「そのエラーはデータベースのマイグレーションが古いのが原因だね。以下のコマンドを試してみて！」

### 3. コマンドの実行指示（Tool Call）
エラーのログを見たい場合や、パッケージをインストールさせたい場合など、**後輩のPC上で直接コマンドを実行させることができます。**

**実行方法**
スレッド内で、**`powershell` または `pwsh`** のMarkdownコードブロックを使ってコマンドを書いて送信してください。

**送信例:**
> 現在のディレクトリ構造を見せてくれる？
> ```powershell
> Get-ChildItem -Path .\src
> ```

**挙動:**
1. あなたがこのメッセージを送信すると、コードブロック内のコマンドが後輩のPC（エディタ）に送信されます。
2. 後輩のPCでコマンドが自動実行されます。
3. 実行が完了すると、その結果（標準出力・エラー文）がBot経由で再びこのスレッドに投稿されます。
4. 結果を見て、さらに指示を出すことができます。

---

## 🔄 システムの全体フローおさらい

1. **後輩**: エディタで質問を送信 ➡️
2. **システム**: Discordに専用スレッドを作成 ➡️
3. **先輩**: スレッドで回答、またはコマンドを指示 ➡️
4. **システム**: 後輩のエディタへ回答をストリーミング（コマンドの場合は実行要求） ➡️
5. **後輩**: (コマンド実行時) 結果をシステムに返送 ➡️
6. **システム**: 結果をDiscordスレッドに投稿 ➡️
7. **先輩**: 結果を見てさらにアドバイス

このループにより、AIのようなシームレスなUIでありながら、裏側では人間の温かいサポートを受けることができます！
